# Checklist de Levantamento de Dados

Use este checklist para entender quais dados o cliente tem, onde eles estão e quais cuidados serão necessários antes de qualquer proposta ou execução.

## 1. Planilhas existentes

- [ ] Existem planilhas de vendas?
- [ ] Existem planilhas de estoque?
- [ ] Existem planilhas de produção, compras ou financeiro?
- [ ] As planilhas seguem um padrão ou cada pessoa usa um modelo diferente?
- [ ] Há colunas com datas, códigos, produtos, clientes, valores e responsáveis bem definidos?
- [ ] Existem abas antigas, duplicadas ou pouco confiáveis?

## 2. Sistemas usados hoje

- [ ] O cliente usa ERP, PDV, sistema de estoque, sistema comercial ou outro software?
- [ ] Qual sistema concentra mais informações importantes?
- [ ] O sistema permite exportar dados?
- [ ] Os dados saem em Excel, CSV, PDF ou outro formato?
- [ ] Existe acesso por usuário ou depende de uma única pessoa?

## 3. Relatórios manuais

- [ ] Quais relatórios são montados manualmente?
- [ ] Quem prepara esses relatórios?
- [ ] Qual o tempo médio gasto por relatório?
- [ ] Existe retrabalho para consolidar números de várias fontes?
- [ ] Há risco frequente de erro ou divergência?

## 4. Frequência de atualização

- [ ] Os dados são atualizados diariamente?
- [ ] Semanalmente?
- [ ] Mensalmente?
- [ ] Existem informações que chegam com atraso?
- [ ] O cliente precisa acompanhar indicadores em tempo real ou em ciclos definidos?

## 5. Responsáveis pelos dados

- [ ] Quem registra os dados na origem?
- [ ] Quem confere a consistência?
- [ ] Quem usa os relatórios para decidir?
- [ ] Existe dependência excessiva de uma pessoa?
- [ ] Se essa pessoa faltar, o processo continua?

## 6. Problemas recorrentes

- [ ] Dados duplicados
- [ ] Informações faltando
- [ ] Códigos diferentes para o mesmo item
- [ ] Datas fora de padrão
- [ ] Erros de digitação
- [ ] Divergência entre relatório e realidade operacional
- [ ] Arquivos espalhados em vários locais

## 7. Indicadores desejados

- [ ] Quais números o cliente gostaria de ver com mais clareza?
- [ ] O que ele precisa acompanhar por dia, semana ou mês?
- [ ] Quais comparações seriam úteis: período, loja, vendedor, produto, máquina ou categoria?
- [ ] O cliente quer apenas acompanhar ou também receber alertas e resumos?

## 8. Riscos e cuidados com dados reais

- [ ] Confirmar se será necessário acessar dados reais nesta fase.
- [ ] Solicitar apenas o mínimo necessário para análise.
- [ ] Evitar compartilhamento informal de dados sensíveis por canais inadequados.
- [ ] Definir quem autoriza o envio dos dados.
- [ ] Anonimizar ou mascarar dados quando possível.
- [ ] Registrar se existem informações pessoais, financeiras ou estratégicas.
- [ ] Combinar prazo e forma de descarte ou devolução dos arquivos recebidos.
